# Open ID Connect

Configuration Rational:

The API Server needs to be configured for trusting a specific CA. 
Because dex will be published by with an ingress, the ssl certificate is used.

If the ingress cert mode is 'CA' (k8s_certs_mode: CA), it is used automatically for the
apiserver 'oidc-ca-file' argument.

If the ingress cert mode is 'ACME', the CA has to be configured as text by 
k8s_certs_ca.certificate, like:
